Methodology

"Top" is two different questions, so the ranking keeps them as two separate dimensions — and shows a blend so you can weigh the tradeoff.

1

Topic relevance frequency

How much a show actually talks about the AI topics you select. We pull each candidate show's episode list and count how many episodes mention your selected phrases in the title or description.

relevance = matching episodes + 0.5 × recent (90d) + density weight × (matching ÷ total)

Source: Podcast Index API — an open podcast database (episode metadata). Requires a free API key.

2

Audience size popularity

How big a show is. We use each show's position in Apple's current Technology & Business podcast charts (0–100, keyless) — the AI-in-business space. Optionally, Listen Notes' Listen Score can be used instead.

audience = Apple Technology/Business chart rank → 0–100  (or Listen Score, if configured)

Sources: Apple Top Podcasts — Technology & Business genre charts (default, no key) · Listen Notes Listen Score (optional).

∑

Blended rank combined

A single weighted score, leaning toward relevance, so you can sort by one number while still seeing both dimensions in the table.

blended = 0.6 × normalized relevance + 0.4 × (audience ÷ 100)

Default sort: relevance when topics are selected; popularity for the overall board. Refined results show big shows only — those present in the Technology/Business charts.

📡

Ad host / DAI ad tech

Which platform hosts & serves ads for each show. We fingerprint the episode audio (enclosure) URL: the host's domain plus any analytics prefixes chained in front.

megaphone.fm → Megaphone · art19.com → Art19 · simplecast → AdsWizz · omny.fm → iHeart · podtrac/chartable → analytics

Source: enclosure URLs from Podcast Index. Identifies the platform (host/DAI + analytics), not the advertiser/DSP; self-hosted shows read as "self-hosted".

Honest limits

  • No real download numbers. True listener/download counts are private; no free API exposes them. The audience column is a modeled popularity proxy, not a download count.
  • Apple's genre charts favor larger shows. A niche-but-on-topic show may rank high on relevance yet show "not in Apple Top chart" for audience — that's expected.
  • Metadata, not transcripts. Relevance is measured from episode titles & descriptions, so a show that discusses a topic without naming it may be undercounted.
